BUGS FIXED (in order of discovery)
в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ

1. VITE_ prefix missing on VAPID public key (.env)
   BEFORE: VAPID_PUBLIC_KEY=04d002...  (Vite never exposes non-VITE_ vars)
   AFTER:  VITE_VAPID_PUBLIC_KEY=BNACgQ5y...
   Effect: urlBase64ToUint8Array(undefined) crashed on .length

2. VAPID public key was hex-encoded, not base64url (.env)
   BEFORE: 04d002810e723eafa1...  (130-char hex string)
   AFTER:  BNACgQ5yPq-hvv2IVjQTOv-Uy5ueGSeQ0FDL-N_89Rrmt9a...
   Derived via py_vapid from the existing PKCS8 private key:
     v = Vapid.from_string(VAPID_PRIVATE_KEY)
     pub = v.public_key.public_bytes(Encoding.X962, PublicFormat.UncompressedPoint)
     base64.urlsafe_b64encode(pub).rstrip(b'=').decode()
   Effect: PushManager.subscribe() threw InvalidAccessError

3. /save-subscription в†’ 404 (wrong URL in frontend)
   BEFORE: apiFetch("/save-subscription", ...)
   AFTER:  apiFetch("/push/save-subscription", ...)
   Reason: pushsubs router is mounted with prefix "/push"

4. Duplicate + broken Pydantic models in pushsubs.py (post-merge)
   The main branch merge left two conflicting SaveSubscriptionRequest
   classes вЂ” first used subscription:dict, second used SubscriptionObject
   but referenced PushKeys which was never defined. Python uses the last
   definition. Dict-style access on a Pydantic model crashes at runtime.
   FIX: Rewrote pushsubs.py cleanly with:
     class PushKeys(BaseModel): p256dh, auth
     class SubscriptionObject(BaseModel): endpoint, keys: PushKeys
     class SaveSubscriptionRequest(BaseModel): userId, subscription: SubscriptionObject
     class RemoveSubscriptionRequest(BaseModel): userId, endpoint
   Route functions updated to dot-notation (data.subscription.endpoint, etc.)
   remove_subscription now uses RemoveSubscriptionRequest (flat: userId + endpoint)
   matching what the frontend sends.

5. Hardcoded fake userId in Settings.jsx
   BEFORE: usePushNotifications("123456789")
   AFTER:  usePushNotifications(user?.id)
   Effect: every subscription saved under user_id="123456789", never found

6. Double mailto: prefix in VAPID claims (notifications.py)
   BEFORE: vapid_claims={"sub": f"mailto:{_VAPID_CLAIMS_EMAIL}"}
           with VAPID_CLAIMS_EMAIL=mailto:fortunenk4jesus@gmail.com
           в†’ produced "mailto:mailto:fortunenk4jesus@gmail.com"
   AFTER:  vapid_claims={"sub": _VAPID_CLAIMS_EMAIL}

7. unsubscribe() was an empty stub (usePushNotifications.js)
   Implemented: get existing subscription via pushManager.getSubscription(),
   call subscription.unsubscribe(), DELETE /push/remove-subscription,
   set isSubscribed=false. Wired into handlePushToggle in Settings.jsx.

8. Push toggle disabled when permission denied (Settings.jsx)
   Added notificationDenied to disabled prop:
     disabled={pushLoading || !user?.id || notificationDenied}

9. VITE_API_WS_URL missing from .env (dev environment)
   The WebSocket URL falls back to wss://csdp490server.qr-manager.net
   when VITE_API_WS_URL is not set. This meant all dev messages went
   to the production backend вЂ” debug prints on localhost never fired.
   FIX: Added VITE_API_WS_URL=ws://localhost:8000 to .env
   NOTE: Remove this line to point back to production.

10. Windows socket errors: WinError 10035 (WSAEWOULDBLOCK)
    All Supabase clients created without HTTP/2 disabled. Under concurrent
    load on Windows, httpx HTTP/2 multiplexing causes socket blocking.
    FIX: Upgraded supabase SDK 2.28.0 в†’ 2.28.3
    Added options=ClientOptions(httpx_client=httpx.Client(http2=False))
    to the create_client() call in pushsubs.py (main pain point).
    NOTE: Other routers (messaging.py, users.py, dependencies.py) still
    use default clients. Apply same fix there if errors reappear.

11. Push toggle resets to OFF on every page refresh
    isSubscribed initialised to false. The hook never checked existing
    browser subscription on mount.
    FIX: Added useEffect in usePushNotifications that calls
    navigator.serviceWorker.getRegistration(swScope) then
    pushManager.getSubscription() to set isSubscribed on mount.
    IMPORTANT: Must use getRegistration(scope) NOT serviceWorker.ready.
    serviceWorker.ready resolves to the SW controlling the CURRENT PAGE.
    In dev mode scope is /src/ but the settings page is at /settings вЂ”
    different scope в†’ wrong registration в†’ getSubscription() returns null.

12. Notifications stopped working after renotify:true was added (sw.js)
    renotify:true requires tag to be a non-empty string or it throws a
    DOMException that event.waitUntil() swallows silently, killing the
    notification entirely.
    FIX: Added tag guard: const tag = payload.tag || "lostlink-default"
    Added .catch((err) => console.error("[SW] showNotification failed:", err))
    to surface future errors.

13. handlePushToggle toggle-off lost in merge (Settings.jsx)
    Post-merge the toggle-off path still had the old TODO stub instead
    of calling unsubscribe(). Re-applied the fix.
